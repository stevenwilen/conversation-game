# Build Game Screen Skill

## Goal

Create mobile-first screens that feel like a polished party game app, not a generic website.

## Rules

- Preserve the locked game rules from CLAUDE.md.
- Keep the UI simple enough to use while passing one phone around a group.
- Prioritize the main card and gesture interactions.
- Use large text, clear spacing, and thumb-friendly layout.
- Avoid dense explanations during gameplay.
- Avoid desktop-first layouts.
- Avoid form-like UI.

## Required Screen Quality

Every game screen should have:

- clear current deck
- clear current depth
- clear card text
- clear gesture hints
- clear next-player transition
- strong mobile spacing
- polished visual hierarchy
- smooth transitions where useful

## Main Gameplay Gestures

- Tap card: answer / stay (also used to continue after the group answers)
- Swipe left: go lighter
- Swipe right: go deeper
- Pivot disabled for the first prototype

## Do Not Add

- scoring
- accounts
- multiplayer
- player profiles
- timers
- risk labels
- tone labels
- topic labels
- extra mechanics not listed in CLAUDE.md
