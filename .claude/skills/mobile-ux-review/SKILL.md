# Mobile UX Review Skill

Review the current app as a mobile-first group conversation game.

## Review Criteria

Check:

- Does it feel like a mobile game instead of a website?
- Is the card the visual focus?
- Can someone read the card easily while holding the phone?
- Are the gestures obvious without big buttons?
- Is tap/swipe behavior clear?
- Is the current depth obvious?
- Is the next action obvious?
- Does the interface avoid clutter?
- Does it work well on iPhone-sized screens?
- Does the next-player transition prevent rapid skipping?
- Does it feel polished enough to test with real people?

## Output Format

Return:

1. What works
2. What feels rough
3. Highest-priority fixes
4. Specific files/components to edit
5. A short improvement plan

Do not rewrite the game mechanics unless the issue is directly caused by the UI.
